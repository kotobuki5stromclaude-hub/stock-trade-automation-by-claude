# 株式売買サポートシステム — 概要

最終更新: 2026-06-30 / 担当: kotob × Claude

## このシステムの目的
楽天証券での株式売買にあたり、Claudeが地合い・テクニカル・各国情勢を自律的に分析・判断し、
テスト運用（ペーパートレード）として売買シミュレーションを行う。
判断根拠・売買履歴はすべてGitHubリポジトリのCSVで管理する。

> ⚠️ Claudeは金融アドバイザーではありません。提示する内容は情報であり、投資助言ではありません。

## 現在のモード: テスト運用（ペーパートレード）
実弾の発注は行わない。市場分析→買い/売り判断→「楽天で約定した想定」でCSVに記録。
楽天証券へのブラウザ操作・ログインは未使用。処理フローとCSV仕様は `運用フロー_workflow.md` を参照。

## 運用プロファイル
- スタイル: スイング（数日〜数週間）中心、IPO重視
- 対象市場: 日本株（個別）/ 米国株（個別）/ 日本IPO / 米国IPO
- リスク許容度: 中間（リスクとリターンのバランス重視）
- 元本想定: 50万円 / ミニ株（単元未満株）前提 / 1銘柄最大5万円
- 執行フロー: Claudeが自律的に分析・判断・CSV記録まで行う。kotobはコメントで介入可能（任意）。

## 自律判断モード（現行）
- 買い/売り/損切り/利確ラインの計算まですべてClaudeが判断・実行する
- kotobのコメントは `アクション候補_actions.md` で受け付けるが、必須ではない
- ポジションサイジングは売買ルール（`売買ルール_trading_rules.md`）に従い自動計算

## システム構成
| 層 | 役割 | 手段 |
|---|---|---|
| 分析・判断 | 地合い・テクニカル・IPO分析・売買決定 | Claude（GitHub Actions経由） |
| 監視・タイミング | 寄り前/大引け前/米国寄り前に自動スキャン | GitHub Actions（cron） |
| 記録 | 売買履歴・判断根拠・ポジション管理 | GitHubリポジトリのCSV |

## ファイル構成
| ファイル | 役割 |
|---|---|
| `README_システム概要.md` | 本ファイル（システム概要） |
| `売買ルール_trading_rules.md` | スタイル/リスク管理/サイジング/損切り・利確ルール |
| `ウォッチリスト_watchlist.md` | 監視銘柄（日本株/米国株/日本IPO/米国IPO） |
| `保有ポジション_positions.md` | 現在の保有状況（スキャン時に自動更新） |
| `トレード日誌_journal.md` | 売買・判断の記録（スキャン時に自動追記） |
| `アクション候補_actions.md` | Claudeの候補提示 / kotobコメント欄 |
| `週次レビュー_weekly_review.md` | 毎週金曜の振り返り記録 |
| `運用フロー_workflow.md` | 処理フローとCSV仕様の詳細 |
| `data/holdings.csv` | **取引台帳（Source of Truth）** append-only。BUY/SELL を別行で追記。過去行は変更しない |

| `data/trade_decisions.csv` | 判断ログ（買い/売り/見送りの根拠） |
| `scripts/scan.py` | 自動スキャンスクリプト本体 |
| `.github/workflows/stock_scan.yml` | GitHub Actions スケジュール定義 |

## 自動スキャンのスケジュール
| 時刻（JST） | 曜日 | 種別 |
|---|---|---|
| 10:00 | 月〜金 | 日本株スキャン（寄り前） |
| 13:00 | 月〜金 | 日本株スキャン（大引け前） |
| 0:00 | 火〜土 | 米国株スキャン（米国寄り前） |
| 16:00 | 金のみ | 週次レビュー |

## セキュリティ方針
- GitHubリポジトリは専用botアカウント `kotobuki5stromclaude-hub` で管理（最小権限）
- `GITHUB_TOKEN` / `ANTHROPIC_API_KEY` はGitHub Actionsのシークレットで管理
- Claudeはパスワード・2FA・認証情報を一切入力しない（固定ルール）
- 実弾の発注・楽天証券へのログインは行わない（テスト運用中）
