# 株式売買サポートシステム — 概要

最終更新: 2026-06-27 / 担当: kotob × Claude

## このシステムの目的
楽天証券での株式売買にあたり、Claudeがテクニカル分析・地政学・各国情勢・市場環境から
判断材料を提供し、kotob のYes/Noで売買を進める。Claudeは「買い/売り推奨」ではなく
判断材料(根拠・シナリオ・リスク・反対意見)を整理して出す。**最終判断は必ず kotob 本人。**

> ⚠️ Claudeは金融アドバイザーではありません。提示する内容は情報であり、投資助言ではありません。


## ⚙️ 現在のモード: テスト運用(ペーパートレード)
実弾の発注は行わない。市場分析→判断→「楽天で約定した想定」で記録、を `data/` のCSVで回す。
楽天証券へのブラウザ操作・ログインは未使用。処理フローとCSV仕様は `運用フロー_workflow.md` を参照。

## 運用プロファイル
- スタイル: スイング(数日〜数週間)中心、IPO重視
- 対象市場: 日本株(個別)/ 米国株(個別)/ 日本IPO / 米国IPO
- リスク許容度: 中間(リスクとリターンのバランス重視)
- 執行フロー: Claudeが分析→提案 → kotobがYes/No → Yesなら Claude が楽天証券の
  発注画面まで入力 → **2FA・最終確定クリックは kotob 本人**
- 認証情報/パスワード/2FA: Claudeは一切入力しない(固定ルール)

## システム3層構造
1. **分析・判断(Claud/クラウド)** … マクロ・地政学・テクニカル・IPO分析 + このファイル群で記憶
2. **監視・タイミング(スケジュール起動)** … 寄り前/大引け前/米国寄り前に自動スキャン→通知
   - リアルタイム板読みは不可。スイング前提なので指値で対応。
3. **執行(kotobのPC + Claude in Chrome)** … 楽天証券ログイン済みChromeを操作。最終クリックは本人。

## ファイル構成
- `README_システム概要.md` … 本ファイル
- `売買ルール_trading_rules.md` … スタイル/リスク管理/サイジング/損切り・利確ルール
- `ウォッチリスト_watchlist.md` … 監視銘柄(日本株/米国株/日本IPO/米国IPO)
- `保有ポジション_positions.md` … 現在の保有・エントリー/損切り/利確/状態
- `トレード日誌_journal.md` … 売買・判断の記録(日付順)

## 保存ルール
- マスター: kotobのPCの共有フォルダ
- ミラー: Claudeクラウド(スケジュール自動スキャン用の読み取りコピー)


## 運用メモ(重要・環境制約)
- このClaude実行環境では `github.com` へのgit通信が塞がれており `git clone/push` は不可。
- 代わりに **GitHub API(api.github.com)経由** でリポを読み書きする(認証はプロキシ注入トークン=連携bot垢 kotobuki5stromclaude-hub)。
- 入出力ヘルパー: `gh_api.py`(get_file / put_file)。新セッションでも `gh_api.py` を取得して使う。
