# 売買ルール / リスク管理

最終更新: 2026-06-27

> ⚠️ 以下の数値は「雛形(たたき台)」です。最終的な数値は kotob 本人が決めて確定してください。
> Claudeは金融アドバイザーではなく、これは投資助言ではありません。

## 基本スタイル
- スイング(数日〜数週間)。デイトレ・スキャルは原則やらない。
- IPO重視(初値・公募価格・需給・主幹事を別途分析)。
- 対象: 日本株(個別)/ 米国株(個別)/ 日本IPO / 米国IPO。
- リスク許容度: 中間。極端な高ボラ一点張り・フルレバは避ける。

## リスク管理(雛形 — 要確定)
- 1トレードあたりの許容損失: 口座総額の【1〜2%】 → 確定値: ____
- 1銘柄あたりの最大ポジション: 口座の【10〜15%】 → 確定値: ____
- 同時保有の上限: 【5〜8銘柄】 → 確定値: ____
- 現金比率: 相場環境で調整(逆風時は厚めに) → 方針: ____

## エントリー条件(例 — 要調整)
- トレンド方向: 日足で上昇トレンド or 明確な底打ち反転。
- 押し目/ブレイク: 移動平均(例 25日/75日)への押し目 or 直近高値ブレイク+出来高増。
- 出来高: 直近平均比で増加を伴う動き。
- ファンダ/テーマ: 決算・材料・テーマ性で説明できること。
- 地合い: マクロ・地政学が極端な逆風でないこと。

## 損切り(雛形 — 要確定)
- テクニカル: 直近スイングロー割れ / 25日線明確割れ など。
- 値幅目安: エントリーから【-7〜8%】を上限 → 確定値: ____
- ルールは事前に決め、入った後に動かさない(損切りを下にずらさない)。

## 利確(雛形 — 要確定)
- リスクリワード: 最低【1:2】を目安 → 確定値: ____
- 分割利確: 半分を利確ライン到達で、残りはトレーリング、など。
- 決算またぎ: 原則【またがない/一部のみ残す】 → 方針: ____

## 執行フロー(固定)
1. Claudeが候補と根拠・想定エントリー/損切り/利確を提示
2. kotob が Yes / No
3. Yes の場合のみ Claude が楽天証券の発注画面まで入力(銘柄/数量/指値/注文種別)
4. 入力内容を kotob が確認 → **2FA・最終確定クリックは kotob 本人**
5. 約定後、`保有ポジション` と `トレード日誌` を更新

## やらないこと(安全ルール)
- パスワード/2FA/認証情報の入力(Claudeは行わない)
- 最終確定クリックの代行(行わない)
- ルール外の衝動的な売買
