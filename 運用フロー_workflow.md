# 運用フロー(テスト運用 / ペーパートレード)

最終更新: 2026-06-27

## モード
**実弾の発注は行わない。** 市場分析にもとづき「楽天証券で約定した想定」でCSVに記録するテスト運用。
楽天証券へのブラウザ操作・ログインは使わない。記録はすべてこのリポの `data/` 配下のCSVで管理。

## データファイル(data/)
- `data/trade_decisions.csv` … なぜ買う/売る判断をしたかの記録(判断ログ)
- `data/holdings.csv` … 取引(取得・売却)の明細。1取引=1行、売却時に同行へ追記
- `data/holdings_summary.csv` … 現在の保有サマリー(銘柄ごとに集約、随時update)

## 処理フロー(8ステップ)
1. 市場・地政学・テクニカル・IPO情報を確認
2. 候補銘柄を `trade_decisions.csv` に記録(判断種別=新規候補/買い判断 等)
3. 買い判断の候補を、楽天証券で買い約定した想定で `holdings.csv` に取得情報を記録(ステータス=保有中)
4. `holdings_summary.csv` の保有情報をupdate
5. 定期的に株価・ニュースを確認
6. 売却判断が出たら `trade_decisions.csv` に理由を記録(判断種別=売り判断)
7. 売り約定した想定で `holdings.csv` の該当取引に売却情報を追記(ステータス=売却済)、損益を確定
8. `holdings_summary.csv` の保有情報をupdate

## 列の入力ガイド(値の揺れ防止・推奨語彙)
- 市場: 東証P / 東証S / 東証G / NYSE / NASDAQ など
- 投資方針(summary): スイング / 中長期 / IPO初動 など
- ステータス(holdings): 保有中 / 売却済
- 判断種別(decisions): 新規候補 / 買い判断 / 売り判断 / 見送り / 保有継続
- 判断(decisions): 買い / 売り / 見送り / ホールド
- 根拠カテゴリ(decisions): テクニカル / ファンダ / マクロ / 地政学 / 需給 / IPO
- 推奨アクション(decisions): 打診買い / 買い増し / 一部利確 / 全売却 / 様子見
- 信頼度(decisions): 高 / 中 / 低
- 日時はISO形式推奨: YYYY-MM-DD HH:MM (JST)
- 取引ID(holdings): T0001 形式の連番

## 注記
- CSVはUTF-8(BOMなし)。Excelで開いて文字化けする場合はBOM付き版を別途出力可。
- 売買はあくまでテスト。実際の資金は動かない。
